# Supporting Students with Course Administration

The **Course Administration** page provides an overview of the courses you teach and gives you tools to support students throughout the semester.

To open Course Administration:

1. From the **Hub Control Panel**, select **Course Administration**.
2. Select a course card to view detailed information about that course.

## Viewing a course

The course details page displays:

- The students enrolled in the course.
- Whether each student's notebook server is currently running.
- When each student last accessed the course.
- Actions for starting, accessing, or stopping a student's server.

![Course Administration details page]({{ site.baseurl }}/assets/images/screenshots/course-administration-details.png){: .figure }

<div class="figure-caption">

## Accessing a student's notebook environment

One of the most useful Course Administration features is the ability to access a student's notebook environment when providing assistance.

To access a student's notebook environment:

1. Click **Start Server**.
2. Wait for the server to start.
3. Click **Access Server**.
4. When you are finished, click **Stop Server**.

![Access Server and Stop Server buttons]({{ site.baseurl }}/assets/images/screenshots/course-administration-access-server.png){: .figure }

<div class="figure-caption">
<strong>Figure 8.</strong> Once a student's server has started, you can access their notebook environment or stop the server when finished.
</div>

{: .good-to-know }
> Opening a student's notebook environment allows you to view the course exactly as the student sees it. This is helpful for office hours, troubleshooting, verifying that course materials are visible, or helping a student locate files or notebooks.

{: .expect }
> Starting a student's notebook server may take a few seconds. Once the server is ready, the **Access Server** button appears automatically.

{: .good-to-know }
> When you are finished viewing a student's notebook environment, click **Stop Server**. This helps conserve shared computing resources and prevents the environment from running unnecessarily.
