# Your JupyterLab Workspace

## At a glance

- Recognize the main parts of JupyterLab.
- Create a new notebook.
- Open an existing notebook.
- Locate your course files.

When your course starts, **JupyterLab** opens automatically.

JupyterLab is where you create, edit, and organize the notebooks and files used in your course.

## The main areas

**File Browser**  
Displays the files and folders in your course. Use it to open existing notebooks and organize course materials.

**Launcher**  
Provides shortcuts for creating notebooks, consoles, terminals, and other file types.

**Notebook**  
Creates a new Jupyter notebook using the Python kernel installed for your course.

**Terminal** *(Optional)*  
Opens a Linux command-line terminal. Most faculty will not need to use the terminal during normal course preparation.

**VS Code** *(Optional)*  
Opens a browser-based Visual Studio Code session for advanced editing and development workflows.

{: .expect }
> The next time you log in, JupyterLab will reopen with your course files available in the File Browser. If you have previously opened notebooks or other files, they may reopen automatically.

{: .good-to-know }
> You do not need to understand every feature of JupyterLab before you begin teaching. Many faculty use only a small portion of JupyterLab's capabilities, and that is perfectly fine.
