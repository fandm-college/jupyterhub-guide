# Recommendations from Research Computing Services

This page collects practical recommendations from Research Computing Services for teaching with F&M JupyterHub.

{: .good-to-know }
> **Request software early.** Software requests are easiest to accommodate when they are included with your initial course request.

{: .good-to-know }
> **Keep a master copy of your course materials.** Store original notebooks, data files, and supporting materials in a consistent location before distributing them to students.

{: .good-to-know }
> **Test notebooks before class.** Open each notebook in F&M JupyterHub and confirm that code runs as expected.

{: .good-to-know }
> **Organize course materials.** Use clear folders and filenames so students can find notebooks, datasets, and supporting files.

{: .good-to-know }
> **Use GitHub and nbgitpuller when you are ready.** GitHub is recommended for managing course materials, and nbgitpuller provides a smooth way to distribute notebooks through Canvas.

{: .good-to-know }
> **Ask for help early.** If you are unsure about software, course setup, or student access, contact Research Computing Services before the course begins whenever possible.
